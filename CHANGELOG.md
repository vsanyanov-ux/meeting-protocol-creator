# История изменений

Все значимые изменения в этом проекте будут фиксироваться в данном файле.

## [5.3.0] — 2026-05-05
### Добавлено
- **📊 Индикатор свободного места**: В футер интерфейса добавлен визуальный индикатор состояния дискового пространства сервера. Теперь пользователи могут видеть доступный объем (в ГБ) и процент заполненности в реальном времени.
- **📈 Мониторинг очереди**: Добавлено отображение количества активных задач в очереди в реальном времени.
- **🎨 Динамический UI**: Цветовая индикация состояния диска (желтый при >70%, красный при >90% заполнения).

## [5.2.0] — 2026-05-05
### Добавлено
- **🛡️ Защита от Disk Fill Attack**: Внедрена проверка общей квоты дискового пространства (`MAX_TOTAL_UPLOADS_SIZE_BYTES`) и лимит на размер очереди (`MAX_QUEUE_SIZE`), что предотвращает отказ в обслуживании.
- **📂 Защита от Path Traversal**: Реализована строгая санитизация имен файлов при сохранении и доступе, исключающая возможность несанкционированного доступа к системным папкам.
- **🧠 Prompt Injection Hardening**: В системные промпты `YandexProvider` и `LocalProvider` добавлены защитные инструкции, предотвращающие манипуляцию логикой ИИ через ввод пользователя.
- **🧟 Автоматическое восстановление очереди**: При старте сервер теперь автоматически находит и сбрасывает "зомби-задачи", прерванные при аварийном завершении.
- **🗄️ Надежность БД (SQLite WAL)**: Включен режим Write-Ahead Logging и атомарные транзакции для предотвращения повреждения базы данных при высоких нагрузках.

### Изменено
- **Безопасность API**: Ужесточены правила CORS и добавлена валидация MIME-типов для всех входящих файлов.
- **Стабильность транзакций**: Все обновления статусов в `StatusManager` теперь выполняются через `BEGIN IMMEDIATE` для исключения блокировок SQLite.

## [5.1.0] — 2026-04-30
### Добавлено
- **Поддержка текстовых документов**: Теперь можно загружать файлы DOCX, PDF и TXT напрямую. Система автоматически извлекает текст и переходит к генерации протокола, минуя этап транскрибации.
- **Улучшенная обработка Email**: 
    - Добавлена детальная диагностика ошибок отправки писем.
    - Ошибки SMTP (таймауты, проблемы авторизации) теперь отображаются в интерфейсе, не прерывая основной процесс.
    - В API добавлен флаг `email_error` для фронтенда.
- **Локализация системных сообщений**: Все статусы обработки и сообщения об ошибках в бэкенде переведены на русский язык для удобства пользователя.

### Изменено
- **Стабильность Langfuse**: Оптимизирована передача контекста сессий и трейсов в Langfuse SDK v4.0.6. Теперь все данные корректно группируются в дашборде.
- **Валидация MIME-типов**: Расширен список поддерживаемых форматов и улучшена точность определения аудио/видео контента.

### Исправлено
- **Критический баг status_updater**: Исправлена ошибка `UnboundLocalError` при загрузке текстовых файлов.
- **Таймауты SMTP**: Устранено «зависание» процесса при блокировке портов SMTP (например, под VPN).

## [5.0.0] — 2026-04-27
### Добавлено
- **Обновленный интерфейс v5.0.0**: 
    - Новые индикаторы контуров безопасности (Закрытый/Открытый) с иконками статуса.
    - Опциональная отправка на Email: добавлен переключатель в интерфейсе.
    - Ускоренная синхронизация: частота обновления статусов увеличена для плавного отображения прогресса.
    - Информационные бейджи: в шапке теперь отображается активная модель и тип контура.
- **Whisper Turbo**: Полный переход на модель `large-v3-turbo`. Скорость транскрипции увеличена в 2.5 раза.
- **Qwen 3.5 (9B)**: Переход на новейшую архитектуру для качественного анализа встреч.
- **Управление VRAM**: Внедрена система «горячей замены» моделей (`keep_alive: 0`) для стабильности на 12 ГБ VRAM.
- **Точные Таймстемпы**: Улучшена навигация по времени в транскрипте.

### Изменено
- **Конфигурация по умолчанию**: Whisper Turbo и Qwen 3.5 установлены как стандартный стек.
- **Архитектура очистки**: Оптимизирован жизненный цикл объектов `faster-whisper` для предотвращения крашей на Windows.

### Исправлено
- **Langfuse Compatibility**: Исправлена ошибка `AttributeError` в Langfuse SDK v4.0.6 путем оптимизации механизма проброса атрибутов сессии.
- **Стабильность длинных файлов**: Исправлен сбой при обработке аудио более 20 минут.

## [4.4.0] — 2026-04-27
### Добавлено
- **Langfuse сессии**: Добавлено персистентное отслеживание сессий (`session_id`) в Langfuse для лучшего мониторинга.

### Изменено
- **Скрытие технической информации**: Технический вывод JSON скрыт из пользовательского интерфейса.
- **Очистка кода**: Удалены временные скрипты и кэш базы данных статусов (SQLite).

## [4.3.0] — 2026-04-24
### Добавлено
- **Универсальные промпты**: Полный переход на нейтральную роль «Профессионального секретаря», что исключило технический уклон и галлюцинации.

### Изменено
- **Формат протокола**: Раздел «НЕРЕШЕННЫЕ ВОПРОСЫ» теперь всегда содержит текст и является финальной точкой документа.
- **Чистота вывода**: Отчет AI-Аудитора теперь используется только для внутренней аналитики.
- **Запрет иноязычных вставок**: Усилены инструкции по предотвращению появления иероглифов в ответах.

## [4.2.1] — 2024-04-24
### Исправлено
- **UI Progress Sync**: Устранена проблема с «откатом» прогресс-бара до 0%.
- **SQLite Persistence**: Окончательный переход на SQLite для хранения статусов.
- **Live Transcription Progress**: Добавлено отображение количества обработанных фрагментов речи.

## [4.2.0] — 2024-04-24
### Добавлено
- **Multi-worker Backend**: Бэкенд переведен на работу через 2 воркера (Uvicorn).
- **Atomic Persistence**: Реализована система атомарного сохранения статуса и чанков.
- **Cross-process GPU Locking**: Файловая блокировка `gpu.lock` для координации ресурсов GPU.
