# История изменений

Все значимые изменения в этом проекте будут фиксироваться в данном файле.

## [5.0.0] — 2026-04-27
### Добавлено
- **Whisper Turbo**: Полный переход на модель `large-v3-turbo`. Скорость транскрипции увеличена в 2.5 раза при сохранении высокого качества.
- **Управление VRAM**: Внедрена система «горячей замены» моделей в видеопамяти (`keep_alive: 0`). Теперь Whisper и LLM не конфликтуют за ресурсы, что исключает ошибки OOM на видеокартах с 12 ГБ VRAM.
- **Диагностические логи**: Расширенное логирование этапов обработки (загрузка моделей, очистка кэша CUDA, прогресс транскрипции) для удобства отладки.

### Изменено
- **Конфигурация по умолчанию**: Whisper Turbo установлен как стандартная модель для локального режима в `.env` и коде.
- **Архитектура очистки**: Оптимизирован жизненный цикл объектов `faster-whisper`, предотвращены Segmentation Faults на Windows-драйверах.

### Исправлено
- **Стабильность длинных файлов**: Исправлен критический сбой при обработке аудиозаписей более 20 минут.
- **Langfuse Attribute Error**: Устранена ошибка инициализации трейсов в Langfuse SDK v4.0.6.

## [4.4.0] — 2026-04-27
### Добавлено
- **Langfuse сессии**: Добавлено персистентное отслеживание сессий (`session_id`) в Langfuse для лучшего мониторинга промптов и генерации в рамках одной встречи.

### Изменено
- **Скрытие технической информации**: Технический вывод JSON скрыт из пользовательского интерфейса. Теперь он используется исключительно для внутренних нужд системы (например, Langfuse) и не засоряет экран пользователя.
- **Очистка кода**: Удалены временные скрипты и кэш базы данных статусов (SQLite), что делает репозиторий более чистым.

### Исправлено
- **Стабилизация диаризации**: Исправлен пайплайн локальной диаризации, решена критическая проблема `NameError: name 'AudioDecoder' is not defined`.

## [4.3.0] — 2026-04-24
### Добавлено
- **Семантическая диаризация**: Интеллектуальное распознавание имен участников (например, "Настя") из контекста беседы, даже если аудио-анализ не дал точных результатов. Теперь система вчитывается в фразы типа «Настя, тебе слово» для определения спикеров.
- **Универсальные промпты**: Полный переход на нейтральную роль «Профессионального секретаря», что исключило технический уклон (инженерия, ГОСТы) и галлюцинации в протоколах на нетехнические темы.

### Изменено
- **Формат протокола**: Раздел «НЕРЕШЕННЫЕ ВОПРОСЫ» теперь всегда содержит текст (или «Не выявлено») и является строго финальной точкой документа. Любой лишний текст после него запрещен.
- **Чистота вывода**: Отчет AI-Аудитора теперь используется только для внутренней аналитики (Langfuse) и больше не встраивается в DOCX-файл, что делает протокол более лаконичным и официальным.
- **Запрет иноязычных вставок**: Усилены инструкции по предотвращению появления китайских иероглифов и английских слов в ответах локальных моделей (Qwen).

### Исправлено
- **Empty Protocol Bug**: Исправлена критическая ошибка синхронизации с Langfuse, из-за которой протокол мог генерироваться пустым при определенных условиях.
- **Скрытие JSON**: Из отчета аудитора (внутри системы) полностью вырезаются блоки JSON со скорингом перед любой визуализацией.

## [4.2.1] — 2024-04-24
### Исправлено
- **UI Progress Sync**: Устранена проблема с «откатом» прогресс-бара до 0% за счет синхронизации имен статусов бэкенда и фронтенда.
- **Status Step Highlighting**: Исправлено отображение активного этапа «Анализ и Саммери» (теперь корректно подсвечивается зеленым со спиннером).
- **SQLite Persistence**: Окончательный переход на SQLite для хранения статусов, что полностью решило проблему блокировок файлов на Windows в многопоточном режиме.
- **Live Transcription Progress**: Добавлено отображение количества обработанных фрагментов речи в реальном времени.

## [4.2.0] — 2024-04-24
### Добавлено
- **Multi-worker Backend**: Бэкенд переведен на работу через 2 воркера (Uvicorn), что обеспечивает 100% отзывчивость интерфейса (статус ONLINE) даже во время тяжелых вычислений.
- **Atomic Persistence**: Реализована система атомарного сохранения статуса и чанков. Обработка длинных записей (1ч+) теперь возобновляема при сбоях.
- **Cross-process GPU Locking**: Файловая блокировка `gpu.lock` для безопасной координации ресурсов видеокарты между процессами.
- **LLM Upgrade**: Официальный переход на модель **Qwen 2.5 7B** (Ollama) для повышения качества анализа и структуры протоколов.

### Исправлено
- **CUDA SegFault**: Устранены критические вылеты при переключении между Whisper и LLM.
- **Long Text Chunking**: Добавлен посимвольный фолбек для обработки текстов без переносов строк.
- **Atomic Status**: Использование `os.replace` для исключения ошибок чтения «битых» файлов статуса.

## [4.1.0] — 2026-04-17

### Добавлено
- **Локальный тюнинг GPU**: Оптимизирована работа с VRAM на видеокартах с 12GB (RTX 3060). Теперь переключение между Whiskey и Ollama происходит без "зависаний" и ошибок выделения памяти.
- **Интеллектуальное контекстное окно**: Для Ollama установлено стабильное окно в 16к токенов (`num_ctx`), что позволяет обрабатывать длинные транскрипты без обрезок.
- **Расширенные таймауты**: Таймаут запросов к локальным LLM увеличен до 15 минут, гарантируя завершение генерации даже для сложных встреч.
- **Прозрачность обработки**: Логирование количества символов/токенов в транскрипте перед отправкой в LLM для лучшей предсказуемости процесса.

### Обновлено
- **Ядро системы**: Переход на CUDA-версию PyTorch (2.6.0+cu124) для корректной очистки кэша видеокарты.
- **Синхронизация**: Версии бэкенда и фронтенда унифицированы до 4.1.0.

## [4.0.0] — 2026-04-16

### Добавлено
- **Корпоративная отказоустойчивость**: Реализована трехуровневая система резервирования оборудования: **GPU (CUDA) → CPU → Облако (Yandex)**. Система автоматически переключает режим обработки при нехватке ресурсов или сбое оборудования.
- **AI-Аудитор 2.0**: Отчет о качестве от Аудитора (полнота, точность, отсутствие галлюцинаций) теперь встраивается напрямую в финальный DOCX-протокол.
- **Корпоративный мониторинг**: Добавлена автоматическая передача оценок (1-5 звезд) от AI-Аудитора в панель управления Langfuse.
- **Pilot Tracker**: Новый инструмент для корпоративных клиентов для отслеживания ROI и метрик качества в рамках пилотных проектов.

### Улучшено
- **Ребрендинг UI/UX**: Унифицирован индикатор статуса системы (Backend/LLM), во всем интерфейсе терминология заменена на **LLM**.
- **Наблюдаемость (Observability)**: Полный переход на **Langfuse SDK v4** со стабильными ID трассировки и детальной отчетностью об ошибках.
- **Производительность**: Оптимизирована загрузка DLL для Faster-Whisper на Windows, исправлены конфликты VRAM при переключении моделей.

### Исправлено
- **Связь**: Устранена ошибка "All connection attempts failed" при взаимодействии с локальным Ollama на Windows.
- **Стабильность бэкенда**: Исправлены критические состояния гонки в логике роутинга между провайдерами.

## [3.1.0] — 2026-04-16
 
### Изменено
- **Ребрендинг**: Проект официально переименован из **PRO-Толк** в **Протоколист**. Обновлена вся документация, айдентика и компоненты интерфейса.
 
## [3.0.0] — 2026-04-15
 
### Добавлено
- **Turbo Mode (GPU Acceleration)**: Полная поддержка NVIDIA CUDA для **Faster-Whisper** и **Ollama**.
- **модель Persistence**: Реализована стратегия "горячих моделей", при которой модели ИИ остаются в VRAM, снижая задержки при повторных запросах.
- **Улучшенный мониторинг**: Добавлена гранулярная трассировка транскрипции в Langfuse для контроля производительности Whisper.
- **Оптимизация под "железо"**: Конвейер настроен для пользовательских GPU (RTX 3060 12GB) и сред WSL2 с ограничениями ресурсов.
 
### Улучшено
- **Скорость конвейера**: Достигнуто **ускорение в 3.5 раза** для локальной обработки (с ~5 мин до <1.5 мин для стандартных записей).
- **Стабильность**: Решены критические проблемы с конкуренцией за VRAM и вылетами по OOM за счет оптимизации жизненного цикла моделей.
- **Интеграция с Docker**: Упрощена логика резервирования GPU в `docker-compose.yml`.
 
### Изменено
- **STT по умолчанию**: Стандартная локальная модель транскрипции заменена на `small` (cuda) для оптимального баланса качества и скорости.

- **Динамические индикаторы**: В интерфейсе добавлены статусы "Local" или "Online" в зависимости от выбранного провайдера.

- **STT Тарификация**: Внедрена логика стоимости на основе длительности записи для логов транскрипции.

## [2.1.1] — 2026-04-10

## [2.1.0] — 2026-04-10
 
### Добавлено

## [2.0.0] — 2026-04-09
 
### Добавлено
- **Офлайн / Локальный режим**: Полная поддержка локального запуска через **Ollama** и **Faster-Whisper**. Интегрирована модель **Qwen 2.5 (3B)** как основной локальный LLM.
- **Гибридный AI-роутер**: Возможность переключения между Yandex Cloud и локальными провайдерами через `.env` или `.bat` скрипты.
- **Мониторинг Langfuse**: Глобальный переход на Langfuse SDK v4 для трассировки всех этапов: транскрипция, генерация, верификация.
- **Оценка стоимости**: Интегрирован расчет затрат и токенов для Яндекс и локальных моделей в личном кабинете мониторинга.
- **AI-Аудитор (Самокритика)**: Автоматизированный механизм верификации, проверяющий протокол на соответствие транскрипции для предотвращения галлюцинаций.
- **Умные таблицы**: Улучшенное форматирование протоколов с автогенерацией таблиц Markdown/Word для задач и решений.
- **Управление средой**: Добавлены `switch_to_local.bat` и `switch_to_yandex.bat` для быстрой настройки.
 
### Исправлено
- **Потеря данных**: Исправлен баг, при котором результаты транскрипции и аудита не сохранялись на финальном этапе генерации.
- **Пропуски в трассировке**: Решена проблема, когда трассы в Langfuse записывались только для части действий.
- **Стабильность локального STT**: Улучшена обработка ошибок и управление ресурсами при длительной транскрипции на CPU/GPU.
- **Ошибки валидации**: Устранены расхождения моделей Pydantic при переключении между провайдерами ИИ.
 
## [1.0.0] - 2026-04-04
 
### Добавлено
- **Основной движок**: Автоматическое резюмирование встреч с использованием YandexGPT и Yandex SpeechKit.
- **Мультиформатность**: Поддержка аудио (MP3, WAV, M4A и др.), видео (MP4, WEBM) и документов (PDF, DOCX, TXT).
- **Профессиональный вывод**: Автоматическое формирование DOCX протоколов со структурами таблиц.
- **AI-Аудитор**: Встроенный механизм проверки точности протокола.
- **UI/UX**: Современный премиальный интерфейс на React с эффектами стекломорфизма и анимациями.
- **Docker Production Ready**: Оптимизированные многоэтапные Docker-файлы для всех сервисов и конфигурация прокси Nginx.
